Documentación UML - API Fondos BTG Pactual

Nota: Este diagrama UML refleja todos los casos de uso principales de la API REST para manejar fondos FPV y FIC de BTG Pactual.

Actor

  • Cliente: Usuario final de la plataforma que puede consultar, suscribirse, cancelar fondos, consultar saldo, actualizar datos y recibir notificaciones.

Casos de Uso (Use Cases)

  • 1) Listar Clientes (GET /api/clientes)
    • Devuelve todos los clientes registrados.
  • 2) Obtener Cliente por ID (GET /api/clientes/{cliente_id})
    • Proporciona información detallada de un cliente específico.
  • 3) Actualizar Cliente (PUT /api/clientes/{cliente_id})
    • Permite actualizar la información de un cliente.
  • 4) Listar Fondos (GET /api/fondos)
    • Devuelve todos los fondos disponibles con su información básica.
  • 5) Ver Detalle de Fondo (GET /api/fondos/{fondo_id})
    • Proporciona información completa de un fondo específico.
  • 6) Crear Fondo (POST /api/fondos)
    • Permite crear un nuevo fondo, definiendo su nombre, monto mínimo y categoría.
  • 7) Actualizar Fondo (PUT /api/fondos/{fondo_id})
    • Permite actualizar información del fondo como monto mínimo o categoría.
  • 8) Suscribirse a un Fondo (POST /api/suscripciones)
    • Permite al cliente realizar la apertura de un fondo, validando saldo y monto mínimo.
    • Genera transacción de tipo "apertura".
    • Envía notificación por email o SMS según preferencia.
  • 9) Cancelar Suscripción (POST /api/suscripciones/cancelar)
    • Permite al cliente retirarse de un fondo y recuperar su inversión.
    • Genera transacción de tipo "cancelación".
  • 10) Listar Suscripciones de un Cliente (GET /api/suscripciones/{cliente_id})
    • Muestra todos los fondos en los que el cliente está actualmente suscrito.
  • 11) Consultar Historial de Transacciones (GET /api/transacciones/{cliente_id})
    • Muestra todas las aperturas y cancelaciones realizadas por el cliente.
  • 12) Consultar Saldo Actual (GET /api/saldo/{cliente_id})
    • Muestra el saldo disponible para nuevas suscripciones a fondos.
  • 13) Enviar Notificación (POST /api/notificaciones)
    • Envía una notificación al cliente vía email o SMS sobre una suscripción o cancelación.

Reglas de Negocio Relevantes

  • El monto inicial del cliente es COP $500.000.
  • Cada transacción genera un identificador único.
  • El cliente solo puede vincularse si tiene saldo suficiente.
  • El retiro de un fondo retorna el valor de vinculación al cliente.
  • Se notifica al cliente vía email o SMS según su preferencia.
  • Se debe respetar el monto mínimo para cada fondo.
