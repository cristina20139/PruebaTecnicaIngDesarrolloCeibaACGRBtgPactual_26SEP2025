Documentación UML - API Fondos BTG Pactual

Nota: Este diagrama UML refleja los casos de uso principales de la API REST para manejar fondos FPV y FIC de BTG Pactual.

Actor

  • Cliente: Usuario final de la plataforma que puede consultar, suscribirse y cancelar fondos.

Casos de Uso (Use Cases)

  • Listar Fondos (GET /api/fondos)
    • Devuelve todos los fondos disponibles con su información básica.
  • Ver Detalle de Fondo (GET /api/fondos/{id})
    • Proporciona información completa de un fondo específico.
  • Suscribirse a un Fondo (POST /api/fondos/{id}/suscribirse)
    • Permite al cliente realizar la apertura de un fondo.
  • Cancelar Suscripción (POST /api/fondos/{id}/cancelar)
    • Permite al cliente retirarse de un fondo y recuperar su inversión.
  • Consultar Historial de Transacciones (GET /api/transacciones)
    • Muestra todas las aperturas y cancelaciones realizadas por el cliente.
  • Consultar Saldo Actual (GET /api/saldo)
    • Muestra el saldo disponible para nuevas suscripciones a fondos.

Cliente --> Cancelar Suscripción

Cliente --> Consultar Saldo Actual

Reglas de Negocio Relevantes

  • El monto inicial del cliente es COP $500.000.
  • Cada transacción genera un identificador único.
  • El cliente solo puede vincularse si tiene saldo suficiente.
  • El retiro de un fondo retorna el valor de vinculación al cliente.
  • Se notifica al cliente vía email o SMS según su preferencia.
